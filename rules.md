# Repository Rules

Read this file at the start of every session before making changes.

## Current Rules

1. Read `ARCHITECTURE.md` before any code change, refactor, new class, or behavior change.
2. Treat `ARCHITECTURE.md` as mandatory. If a request conflicts with it, explain the conflict instead of implementing the change.
3. Keep this file updated when the user adds a new repository-specific rule that future sessions must remember.
